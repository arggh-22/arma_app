**2. Create `lib/features/log/data/services/log_service.dart`** (D-10, D-11):
```dart
import 'dart:async';
import 'dart:io';
import 'package:flutter/foundation.dart';
import 'package:path_provider/path_provider.dart';
import 'package:share_plus/share_plus.dart';

/// Ring buffer log service for Xray-core debug events.
/// Keeps last [maxLines] lines in memory, streams new lines for live viewing,
/// and exports all buffered lines to a file for sharing.
class LogService {
  static const int maxLines = 5000;

  final _buffer = <String>[];
  final _controller = StreamController<String>.broadcast();

  /// Stream of new log lines for live viewing (MON-05).
  Stream<String> get logStream => _controller.stream;

  /// All currently buffered lines.
  List<String> get lines => List.unmodifiable(_buffer);

  /// Number of buffered lines.
  int get lineCount => _buffer.length;

  /// Add a log line with timestamp prefix.
  void addLine(String line) {
    final timestamp = DateTime.now().toIso8601String().substring(11, 19); // HH:MM:SS
    final timestamped = '[$timestamp] $line';
    _buffer.add(timestamped);
    if (_buffer.length > maxLines) {
      _buffer.removeAt(0);
    }
    _controller.add(timestamped);
  }

  /// Export all buffered lines to a text file and share via system share sheet (MON-06).
  Future<void> exportAndShare() async {
    if (_buffer.isEmpty) return;
    final dir = await getApplicationDocumentsDirectory();
    final timestamp = DateTime.now().millisecondsSinceEpoch;
    final file = File('${dir.path}/arma_vpn_log_$timestamp.txt');
    await file.writeAsString(_buffer.join('\n'));
    await SharePlus.instance.share(
      ShareParams(
        files: [XFile(file.path)],
        subject: 'Arma VPN Logs',
      ),
    );
  }

  /// Clear all buffered lines.
  void clear() {
    _buffer.clear();
  }

  /// Dispose the stream controller.
  void dispose() {
    _controller.close();
  }
}
```

Note on `share_plus`: Check the actual API of share_plus 12.x. If `SharePlus.instance.share(ShareParams(...))` is not the correct API, use the alternative: `Share.shareXFiles([XFile(file.path)], subject: 'Arma VPN Logs')`. Read the share_plus documentation at runtime to determine the correct API.

**3. Create `lib/features/log/presentation/providers/log_provider.dart`:**
```dart
import 'dart:async';
import 'package:riverpod_annotation/riverpod_annotation.dart';
import 'package:arma_proxy_vpn_client/features/log/data/services/log_service.dart';
import 'package:arma_proxy_vpn_client/features/connection/data/datasources/vpn_platform_service.dart';

part 'log_provider.g.dart';

/// Singleton LogService instance.
@Riverpod(keepAlive: true)
LogService logService(Ref ref) {
  final service = LogService();

  // Subscribe to debug events from VPN EventChannel
  // The existing EventChannel already streams {"type": "debug", "message": "..."} events
  // (see Pitfall 7 in RESEARCH.md — debug events ALREADY flow, just filter them)
  final vpnService = VpnPlatformService();
  final subscription = vpnService.vpnEvents
      .where((event) => event['type'] == 'debug')
      .listen((event) {
    final message = event['message'] as String? ?? '';
    if (message.isNotEmpty) {
      service.addLine(message);
    }
  });

  ref.onDispose(() {
    subscription.cancel();
    service.dispose();
  });

  return service;
}

/// Reactive log lines list for the log viewer.
/// Rebuilt when new lines arrive via stream subscription.
@Riverpod(keepAlive: true)
class LogLinesNotifier extends _$LogLinesNotifier {
  StreamSubscription<String>? _subscription;

  @override
  List<String> build() {
    final service = ref.watch(logServiceProvider);
    _subscription = service.logStream.listen((_) {
      // Rebuild state with current buffer contents
      state = service.lines;
    });

    ref.onDispose(() => _subscription?.cancel());

    return service.lines;
  }
}
```

```
┌─────────────────────────────────┐
```

Implementation:
- `ConsumerStatefulWidget` with `ScrollController` for auto-scroll detection
- **AppBar:** Back button, title `l10n.viewLogs`, action: `IconButton(Icons.upload_file)` triggering `logServiceProvider.exportAndShare()`
- **Filter bar:** Row with `DropdownButton` (options: All/Info/Warning/Error from l10n keys `logFilterAll`/`logFilterInfo`/`logFilterWarning`/`logFilterError`) and `TextField` with `Icons.search` prefix for substring search. Wrap in `SingleChildScrollView(scrollDirection: Axis.horizontal)` for narrow screens.
- **Log list:** `ListView.builder` with itemCount from filtered log lines. Each line uses `Text` with `TextStyle(fontFamily: 'monospace', fontSize: 12)`, `bodySmall` weight. Color-code: lines containing `"err"` or `"error"` (case-insensitive) use `colorScheme.error`, lines containing `"warn"` or `"warning"` use `Colors.orange`, others use default `onSurface`. Timestamp prefix (text before first `]`) in `colorScheme.onSurfaceVariant`.
- **Auto-scroll:** When enabled (default ON), scroll to bottom after each new line. Disable when user scrolls up (detect via `ScrollController.position.pixels < ScrollController.position.maxScrollExtent - 50`). Show toggle in status bar with `Switch` widget.
- **Status bar:** Bottom bar with `labelSmall` line count (`l10n.linesCount(count)`) and auto-scroll `Switch`.
- **Filter logic:** Filter lines by: (1) selected level (all=show everything, info=exclude warn/error, warning=only warn, error=only error), (2) search text substring match. Apply both filters.
- **Empty state:** When no lines, show centered text `l10n.noLogsYet` with `l10n.noLogsBody` subtitle.
- **Padding:** 16px horizontal for log lines, 8px vertical for filter bar.
