# Phase 3 — UI Design Contract

> Visual and interaction contract for Subscriptions & Server Intelligence. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Flutter Material 3 (existing) |
| Preset | `ColorScheme.fromSeed(seedColor: Color(0xFF00897B))` — teal seed |
| Component library | Material 3 widgets (`Card`, `NavigationBar`, `AlertDialog`, `FilledButton`, etc.) |
| Icon library | Material Icons (built-in `Icons.*`) |
| Font | System default (Roboto on Android) via Material 3 TextTheme |

**Source:** `lib/core/theme/app_theme.dart` — established in Phase 1, no changes needed.

---

## Spacing Scale

Declared values (must be multiples of 4, from standard set {4, 8, 16, 24, 32, 48, 64}):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, gap between header/first card |
| sm | 8px | Compact element spacing, card internal gaps, mini-FAB stack spacing, compact card vertical padding, stat card padding |
| lg | 16px | Default card padding, horizontal screen margins, section gaps |
| xl | 24px | Between server groups, major section breaks |
| 2xl | 32px | Empty state horizontal padding, page-level content inset |
| 3xl | 48px | Not used this phase |
| 4xl | 64px | Empty state icon size |

Exceptions: Touch targets must be minimum 48×48dp per Material accessibility guidelines. Mini-FAB options already use 40×40dp (acceptable per M3 spec for secondary actions).

**Functional constants (not design tokens):**

| Constant | Value | Usage |
|----------|-------|-------|
| `kFabClearance` | 88px | Bottom list padding to clear the FAB and mini-FAB stack |

**Source:** Established spacing from Phase 1 `ServerCard` (16px padding), `ServerGroupHeader` (16px horizontal, 8px vertical), `ImportFab` (8px stack spacing), `EmptyServerState` (16px/24px/32px).

## Typography

All typography uses Material 3 `TextTheme` roles — no custom font sizes. This ensures automatic scaling with system accessibility settings and Dynamic Type.

| Role | M3 TextTheme | Size | Weight | Line Height | Usage in Phase 3 |
|------|-------------|------|--------|-------------|-------------------|
| Body | `bodyMedium` | 14sp | 400 (regular) | 1.43 (20sp) | Server address, subscription data info, log lines, filter labels |
| Body small | `bodySmall` | 12sp | 400 (regular) | 1.33 (16sp) | Share link preview text, helper text, error messages below fields |
| Label | `labelMedium` | 12sp | 500 (medium) | 1.33 (16sp) | Protocol badges, latency values, sort/filter chip text, timestamp prefixes |
| Label small | `labelSmall` | 11sp | 500 (medium) | 1.45 (16sp) | Log viewer line count, metadata captions |
| Title | `titleMedium` | 16sp | 500 (medium) | 1.5 (24sp) | Server name, subscription group header name, dialog titles |
| Headline | `headlineSmall` | 24sp | 400 (regular) | 1.33 (32sp) | Empty state headings, log viewer screen title |

Additional role for log viewer only:

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Monospace | `monospace` fontFamily | 12sp | 400 | 1.5 (18sp) | Log viewer lines — must use monospace for alignment |

**Source:** Established from Phase 1/2 — `ServerCard` uses `titleMedium`/`bodyMedium`, `ProtocolBadge` uses `labelMedium`, `EmptyServerState` uses `headlineSmall`/`bodyMedium`.

## Color

This phase inherits the Material 3 dynamic color scheme from `ColorScheme.fromSeed(seedColor: Color(0xFF00897B))`. All colors are derived from the scheme, never hardcoded hex values (except protocol badges which are already established).

| Role | Token | Light Value | Dark Value | Usage |
|------|-------|-------------|------------|-------|
| Dominant (60%) | `colorScheme.surface` | #FAFAFA-ish | #1C1B1F-ish | Scaffold background, list background |
| Secondary (30%) | `colorScheme.surfaceContainerLow` | #F3F3F3-ish | #252529-ish | Cards (server cards, stat cards, subscription headers), dialog surfaces |
| Accent (10%) | `colorScheme.primary` | #00897B (teal) | #4DB6AC (lighter teal) | See reserved-for list below |
| Destructive | `colorScheme.error` | #BA1A1A | #FFB4AB | Delete actions, failed latency, error states |

**Accent reserved for:**
1. Selected server card border (2px)
2. Subscription group header name text
3. Active sort/filter indicator (selected chip fill)
4. "Best Server" button fill
5. FAB and mini-FAB backgrounds
6. QR scanner crosshair overlay frame

**Latency color scale (semantic, inline in server card):**

| Range | Color | Token |
|-------|-------|-------|
| 0–150ms | Green | `Colors.green` (0xFF4CAF50) |
| 151–300ms | Orange/Amber | `Colors.orange` (0xFFFF9800) |
| 301ms+ | Red | `colorScheme.error` |
| Testing… | Muted | `colorScheme.onSurfaceVariant` with progress indicator |
| Failed (−1) | Red + icon | `colorScheme.error` with `Icons.error_outline` |

**Multi-select highlight:**
- Selected cards: `colorScheme.primaryContainer` background tint
- Checkbox: `colorScheme.primary` when checked

**Source:** `app_theme.dart` seed color, `ServerCard` selection border pattern, `AppColors` protocol badge colors.

**File:** `lib/features/server/presentation/widgets/server_card.dart`

#### 4. ServerListScreen (extend)

**File:** `lib/features/server/presentation/screens/server_list_screen.dart`

**Focal point:** The server card list is the primary visual focus. On first load, the list occupies 70%+ of viewport height with no competing elements above the fold. The SortFilterBar is visually recessive (muted chip colors, no borders) so it doesn't compete with card content.

**New elements:**
- **Sort/Filter bar:** Horizontal row below AppBar. Contains sort dropdown + filter chips.
- **Multi-select AppBar:** When multi-select is active, AppBar transforms: title shows `"{count} selected"`, leading shows `IconButton(Icons.close)` to exit selection, actions show `[Select All]` and `[Delete]` icons.
- **Pull-to-refresh:** `RefreshIndicator` wrapping the ListView. Triggers subscription auto-update for all subscriptions.
- **"Best Server" button:** In AppBar actions (normal mode), `IconButton(Icons.auto_awesome)` — runs auto-select algorithm. Disabled (greyed) if no latency data.
- **"Test All" button:** In AppBar actions (normal mode), `IconButton(Icons.speed)` with `Semantics(label: 'Test all servers')` — triggers bulk latency test.

**AppBar (normal mode):**
```

**AppBar (multi-select mode):**
```

### New Components

#### 5. SortFilterBar

**File:** `lib/features/server/presentation/widgets/sort_filter_bar.dart`

**Layout:**
```
Sort: [Latency ▼]   [All] [Working] [Failed]
```

- **Sort dropdown:** `DropdownButton` with options: `Name`, `Latency`, `Protocol`. Default: `Name`. Shows `Icons.sort` prefix icon. `bodyMedium` text.
- **Filter chips:** `FilterChip` widgets. Options: `All` (default, filled), `Working` (latency ≤ 300ms), `Failed` (latency > 300ms or −1). Single-select — tapping one deselects others.
- **Height:** 48dp row.
- **Padding:** 16px horizontal, 8px vertical.
- **Scrollable:** Wrap in `SingleChildScrollView(scrollDirection: Axis.horizontal)` for narrow screens.

#### 6. LatencyIndicator

**File:** `lib/features/server/presentation/widgets/latency_indicator.dart`

**States:**

| State | Display | Style |
|-------|---------|-------|
| Untested | `"—"` | `labelMedium`, `onSurfaceVariant` |
| Testing | `SizedBox(16, 16, CircularProgressIndicator(strokeWidth: 2))` | `primary` color |
| Good (0-150ms) | `"87ms"` | `labelMedium`, `Colors.green` |
| Fair (151-300ms) | `"234ms"` | `labelMedium`, `Colors.orange` |
| Poor (301ms+) | `"502ms"` | `labelMedium`, `colorScheme.error` |
| Failed | `Icons.error_outline` (16dp) | `colorScheme.error` |

**Tap behavior:** Tapping the indicator re-tests that individual server (SERV-03).

**Width:** Constrained to 56dp to prevent layout shift when values change.

#### 7. QrScannerScreen

**File:** `lib/features/server/presentation/screens/qr_scanner_screen.dart`

**Full-screen camera view with:**
- **MobileScanner widget** filling entire screen
- **Overlay:** Semi-transparent black (`Colors.black54`) with a 250×250dp transparent cutout in center, bordered by 2px `colorScheme.primary` rounded rectangle (12px radius)
- **AppBar:** Transparent, overlaid on camera. Back button (`Icons.arrow_back`), title "Scan QR Code"
- **Flash toggle:** Bottom-right, `IconButton(Icons.flash_on / Icons.flash_off)`, 48dp touch target, white icon on dark scrim
- **Camera switch:** Bottom-left, `IconButton(Icons.cameraswitch)`, 48dp touch target (only if device has multiple cameras)
- **Bottom instruction:** `"Point camera at a QR code"` — `bodyMedium`, white text, centered below cutout, 16px below cutout bottom

**Auto-detect behavior (D-07):**
- Share link detected → auto-import, pop screen, show success snackbar
- HTTP(S) URL detected → show dialog: "This looks like a subscription URL. Add as subscription?" with [Not Now] [Add Subscription] buttons
- Unrecognized content → show error snackbar: "Couldn't recognize QR code content"

**Navigation:** Pushed as a MaterialPageRoute (not a tab). Back returns to server list.

## Interaction Contracts

### Multi-Select Mode (SERV-05)

| Trigger | Action |
|---------|--------|
| Long-press on any server card | Enter multi-select mode; first card auto-selected |
| Tap server card (in multi-select) | Toggle selection (checkbox on/off) |
| Tap "Select All" in AppBar | Select all visible servers |
| Tap "✕" (close) in AppBar | Exit multi-select, clear selection |
| Tap "Delete" in AppBar | Show bulk delete confirmation dialog |
| Navigate away from tab | Exit multi-select, clear selection |

**Bulk delete confirmation dialog:**
```
Delete {count} servers?

This will permanently remove the selected servers.
Servers from subscriptions will reappear on next refresh.

                [Keep Servers]   [Delete {count}]
```
- "Keep Servers" button: `TextButton`, dismisses dialog without deleting
- "Delete {count}" button: `TextButton` with `colorScheme.error` foreground

### Latency Testing (SERV-03, SERV-04)

| Trigger | Action |
|---------|--------|
| Tap latency indicator on a card | Test that single server |
| Tap "Test All" in AppBar | Bulk test all servers (3 concurrent, progressive update) |
| Bulk test in progress | AppBar "Test All" button shows `CircularProgressIndicator` (20dp), disabled |
| Single test in progress | That card's latency indicator shows spinner |
| Test complete (success) | Indicator shows `"XXms"` color-coded |
| Test complete (failure) | Indicator shows `✕` error icon |

### Pull-to-Refresh (D-04)

| Trigger | Action |
|---------|--------|
| Pull down on server list | `RefreshIndicator` activates, refreshes ALL subscriptions |
| Refresh in progress | Standard Material pull-to-refresh spinner |
| Refresh complete | Snackbar: "Updated {count} subscriptions" or "All up to date" |
| Refresh failed (one sub) | Snackbar: "Failed to update {name}. Check your connection." |

### Subscription Collapse (D-12)

| Trigger | Action |
|---------|--------|
| Tap collapse chevron on subscription header | Collapse: hide all server cards under this subscription. Chevron rotates to `expand_more`. |
| Tap expand chevron | Expand: show all server cards. Chevron rotates to `expand_less`. |
| Default state | Expanded (all servers visible) |
| Animation | `AnimatedCrossFade` or `AnimatedSize` with 200ms duration |

### QR Scanner (D-07)

| Trigger | Action |
|---------|--------|
| Detect share link QR | Auto-import server, pop scanner, show success snackbar |
| Detect HTTP(S) URL QR | Show dialog: "Add as subscription?" with [Not Now] [Add Subscription] |
| Detect unknown content | Error snackbar: "Couldn't recognize this QR code" |
| Tap flash toggle | Toggle camera flash on/off |
| Tap back button | Pop scanner, return to server list |

### Config Export/Share (D-18)

| Trigger | Action |
|---------|--------|
| Long-press server card → "Share" (context menu) | Open `QrDisplayDialog` modal bottom sheet |
| Tap "Copy Link" | Copy share link to clipboard, snackbar: "Link copied" |
| Tap "Share Link" | Open system share sheet with share link text |

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Screen reader labels | All interactive elements have `Semantics` labels (follow existing `ServerCard` pattern). "Test All" button uses `Semantics(label: 'Test all servers')`. |
| Touch targets | Minimum 48×48dp for all tappable areas |
| Color contrast | Material 3 `ColorScheme.fromSeed` guarantees WCAG AA for text on surface tokens |
| Latency colors | Never rely on color alone — always include numeric text alongside color |
| Multi-select state | Checkboxes provide non-color state indication |
| QR scanner | Fallback: manual URL input via Add Subscription dialog for users who cannot use camera |
| RTL support | All layouts use `start`/`end` alignment (never `left`/`right`). Persian locale tested. |
