## Visual Hierarchy Contract

Guide screen visual priority order:
1. Primary focus: `Link` submit area state (idle/loading/success/failure feedback) as the core phase-13 hardening target.
2. Secondary focus: `Open Telegram Bot` CTA and step guidance context.
3. Tertiary focus: supporting copy and passive informational text.

This order is mandatory and should remain visually obvious through spacing, contrast, and interaction affordances.

---

## Interaction & State Contract

1. **Submit idle**
   - `Link` button enabled, `Paste` enabled, input enabled.

2. **Submit in-flight (`isSubmitting=true`)**
   - Disable input, `Paste`, and `Link` button.
   - Show inline spinner inside `Link` button (existing pattern).

3. **Success (`linked`)**
   - Show success snackbar text.
   - Auto-pop back to Dashboard.

4. **Recoverable failures (`alreadyLinked`, `invalidId`, `network`, `server`, `unknown`, `unauthorized`)**
   - Stay on screen.
   - Show mapped snackbar message.
   - Re-enable all controls immediately after response.
   - Retry path is same primary `Link` action (no secondary dialog required).

5. **Validation hardening**
   - Trim input before submit.
   - Accept only numeric ID with length 5–20.
   - On invalid value, no network request.

6. **Duplicate-tap hardening**
   - Must reuse provider in-flight guard (single active submit future).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `Link Telegram` |
| Empty state heading | `No Telegram ID yet` |
| Empty state body | `Open the bot, run Get Telegram ID or /my_id, then paste your ID to continue.` |
| Error state | `Couldn’t link Telegram right now. Check your ID and connection, then try again.` |
| Destructive confirmation | none in this phase |

Outcome copy mapping (must match app localization tone):
- linked → `Telegram account linked successfully.`
- alreadyLinked → `This Telegram account is already linked.`
- invalidId → `Telegram ID is invalid. Use 5–20 digits.`
- unauthorized → `Session expired. Please sign in again, then retry linking.`
- network → `No network connection. Reconnect and try again.`
- server → `Server error. Please try again shortly.`
- unknown → `Unexpected error. Please try again.`

## Validation Coverage Contract

Minimum required for Phase 13 completion:

1. **Widget tests**
   - In-flight disables input/paste/submit and shows spinner.
   - Success path shows feedback and pops route.
   - Each failure outcome shows correct message and remains on screen.
   - Retry works after a failure (controls re-enabled).

2. **Provider/unit tests**
   - Validation rejects non-digit and out-of-range IDs.
   - Duplicate submit returns in-flight future (no duplicate repository calls).
   - Unknown exception fallback maps to `unknown`.

3. **UX regression checks**
   - Dashboard FAB still navigates to Telegram guide.
   - Scroll hide/show FAB behavior preserved.
