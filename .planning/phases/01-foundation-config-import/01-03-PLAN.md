  <implementation>
Follow RED→GREEN→REFACTOR cycle strictly per D-12 (pure Dart, unit-testable):

**RED phase:** Create test files first. Write all test cases listed above using `flutter_test`. Each parser gets its own test file. Tests will fail because parser files don't exist yet.

**GREEN phase:** Implement parsers one at a time:

1. **`share_link_parser.dart`** — Static `parse(String input)` method:
   - Trim input
   - Check scheme prefix → dispatch to specific parser
   - Try raw JSON parsing as fallback (for CONF-06 support)
   - Return `ServerConfig?` (null on failure)

2. **`vless_parser.dart`** — Static `parse(String uri)`:
   - Use `Uri.parse(uri)` to extract host, port, userInfo (uuid), queryParameters, fragment (name)
   - Map query params: `type`→network, `security`→security, `sni`→sni, `pbk`→publicKey, `fp`→fingerprint, `sid`→shortId, `spx`→spiderX (Uri.decodeComponent), `flow`→flow, `path`→path, `host`→host, `serviceName`→serviceName
   - Generate UUID for `id` field using `uuid` package
   - Set `addedAt: DateTime.now()`
   - Wrap in try-catch, return null on any exception

3. **`vmess_parser.dart`** — Static `parse(String uri)`:
   - Strip `vmess://` prefix
   - **Format detection per Pitfall 1:** If remaining content contains `@` → standard URI format. Otherwise → legacy base64-JSON.
   - **Legacy path:** Normalize base64 (replace `-`→`+`, `_`→`/`, pad with `=`), decode, parse JSON. Extract: `ps`→name, `add`→address, `port`→port (parse as int), `id`→uuid, `aid`→alterId, `scy`→encryption, `net`→network, `tls`→security ("tls"→"tls", ""→"none"), `sni`→sni, `host`→host, `path`→path
   - **Standard URI path:** Use `Uri.parse()` like VLESS
   - Both paths produce ServerConfig with `protocol: ProtocolType.vmess`

4. **`trojan_parser.dart`** — Static `parse(String uri)`:
   - `Uri.parse(uri)` — userInfo is the password (URL-decode it)
   - Map same query params as VLESS (type, security, sni, host, path, etc.)
   - Default security to "tls" if not specified (Trojan convention)

5. **`shadowsocks_parser.dart`** — Static `parse(String uri)`:
   - Strip `ss://` prefix
   - Split on `@` — left side is base64-encoded `method:password`, right side is `server:port`
   - Decode base64 left side, split on first `:` to get method and password
   - Handle URL-safe base64 and missing padding
   - Extract fragment for name

6. **`hysteria2_parser.dart`** — Static `parse(String uri)`:
   - Handle both `hysteria2://` and `hy2://` schemes
   - userInfo is auth/password
   - Query params: `sni`, `insecure`, `obfs`, `obfs-password`

**REFACTOR phase:** Extract shared URI parsing utility if pattern emerges across parsers.

All parsers: use `import 'package:uuid/uuid.dart';` for ID generation, use `dart:convert` for base64, wrap ALL parsing in try-catch returning null on failure.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | Tampering | All parsers | mitigate | Wrap all parsing in try-catch, return null on ANY exception. Validate required fields (address non-empty, port 1-65535) before constructing ServerConfig. Use Dart's Uri.parse() which is memory-safe. |
| T-01-03-02 | Tampering | vmess_parser.dart | mitigate | Validate base64 content before JSON decode. Catch FormatException from base64.decode. Catch FormatException from json.decode. Never trust `v` field value — attempt parse regardless. |
| T-01-03-03 | Denial of Service | All parsers | mitigate | Limit input string length to 10000 characters before parsing (share links are typically <500 chars). Truncate server name to 50 chars (AppConstants.maxServerNameLength) to prevent UI overflow. |
| T-01-03-04 | Tampering | shadowsocks_parser.dart | mitigate | Validate method string against known SS encryption methods. Reject unknown methods with null return. |
