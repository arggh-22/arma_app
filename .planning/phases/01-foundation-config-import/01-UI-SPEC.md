## Design System

| Property | Value |
|----------|-------|

## Spacing Scale

All values in logical pixels (dp). Based on 4-point grid, Material 3 conventions.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4dp | Icon-to-label gaps, inline badge padding |
| sm | 8dp | Compact element spacing, chip internal padding |
| base | 16dp | Default content padding, card internal padding, list item internal padding, FAB spacing |
| lg | 24dp | Section gaps within a screen, between card groups |
| xl | 32dp | Bottom nav height padding, major layout gaps |
| 2xl | 48dp | Dashboard connect button outer margin |
| 3xl | 64dp | Not used in Phase 1 |

Exceptions:
- Bottom navigation bar height: 80dp (Material 3 default with labels)
- FAB size: 56dp (Material 3 default)
- Mini FAB options: 40dp (expanded FAB sub-items)
- Touch targets: minimum 48dp × 48dp (Material accessibility guideline)

## Typography

Using Material 3 `TextTheme` from `ThemeData`. Roboto (system default). Exactly 4 roles used in Phase 1:

| Role | Size | Weight | Line Height | M3 TextStyle |
|------|------|--------|-------------|--------------|
| Body | 14dp | 400 (regular) | 1.43 (20dp) | `bodyMedium` |
| Label | 12dp | 500 (medium) | 1.33 (16dp) | `labelMedium` |
| Title | 16dp | 500 (medium) | 1.5 (24dp) | `titleMedium` |
| Heading | 22dp | 400 (regular) | 1.27 (28dp) | `headlineSmall` |

Additional constrained usage:
- Protocol badge text: `labelMedium` — 12dp, weight 500 (same tier as bottom nav labels)
- Bottom nav labels: `labelMedium` — 12dp, weight 500
- FAB labels: `labelLarge` — 14dp, weight 500
- App bar title: `titleLarge` — 22dp, weight 400

**Weight constraint:** Only 2 weights used across entire Phase 1: **400 (regular)** and **500 (medium)**. No bold (700), no light (300).

## Color

### Seed Color

`Color(0xFF00897B)` — Teal 600. All palettes generated via `ColorScheme.fromSeed()`.

**Source:** D-01 from CONTEXT.md — "Teal/cyan accent — modern feel, differentiated from competitors' blue/indigo schemes"

### Light Theme

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `surface` — #FFFBFE (near-white) | Screen backgrounds, scaffold |
| Secondary (30%) | `surfaceContainerLow` — M3 generated | Cards, bottom nav background, server group headers |
| Accent (10%) | `primary` — Teal (M3 generated from seed) | See reserved-for list below |
| Destructive | `error` — M3 generated (#BA1A1A) | Delete server, destructive confirmations only |

### Dark Theme

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `surface` — M3 dark (near-black, ~#1C1B1F) | Screen backgrounds, scaffold |
| Secondary (30%) | `surfaceContainerLow` — M3 dark generated | Cards, bottom nav background, server group headers |
| Accent (10%) | `primary` — Teal (lighter M3 dark variant) | See reserved-for list below |
| Destructive | `error` — M3 dark generated | Delete server, destructive confirmations only |

### Accent Reserved For (explicit list — these elements only)

1. **Active bottom nav icon + label** — selected tab indicator
2. **FAB background** — the expandable import FAB on Servers screen
3. **Selected server card border/highlight** — the currently active server
4. **Connect button fill** — the placeholder circular button on Dashboard
5. **Switch/toggle active state** — theme toggle, routing toggles
6. **Protocol badge background** — colored chips on server cards (see Protocol Badge Colors below)
7. **Snackbar action text** — "Undo", "Retry" in snackbars

### Protocol Badge Colors

Each protocol gets a distinct chip color for instant visual identification. Colors are derived from Material 3 palette, NOT from the teal seed (to ensure differentiation):

| Protocol | Background | Text | Rationale |
|----------|-----------|------|-----------|
| VLESS | `Color(0xFF00897B)` Teal 600 | White | Primary protocol, matches app accent |
| VMess | `Color(0xFF1565C0)` Blue 800 | White | Legacy V2ray — blue is traditional |
| Trojan | `Color(0xFFE65100)` Orange 900 | White | Warning-adjacent — trojan = infiltration |
| SS | `Color(0xFF6A1B9A)` Purple 800 | White | Shadowsocks — distinctive, no semantic conflict |
| Hysteria2 | `Color(0xFF2E7D32)` Green 800 | White | Fast/go — green = speed association |

## Screen Contracts

### Screen A: Dashboard

**Purpose:** Home screen. In Phase 1, shows placeholder connect button (non-functional) and active server info. VPN connection is Phase 2.

**Focal point:** Connect button — primary visual anchor, centered in upper-middle of screen.

| Element | Specification |
|---------|--------------|
| App bar | Title: "Arma VPN" — `titleLarge`, no back button, settings gear icon (trailing) removed (settings is a tab) |
| Connect button | Centered circular button, 120dp diameter, teal `primary` fill, white power icon (`Icons.power_settings_new`), `onPrimary` color. **Phase 1: disabled state** — 50% opacity, tap shows snackbar "Connection available in next update" |
| Connection state label | Below connect button, 16dp gap. Text: "Not Connected" — `titleMedium`, `onSurfaceVariant` color |
| Active server card | Below state label, 24dp gap. Card with 12dp radius, `surfaceContainerLow` fill. Shows selected server name + protocol badge. If no server selected: "No server selected" in `bodyMedium`, `onSurfaceVariant` |
| Traffic stats placeholder | Below active server card, 16dp gap. Two columns: "↓ 0 B/s" and "↑ 0 B/s" — `bodyMedium`, `onSurfaceVariant`. Static placeholder for Phase 1. |

### Screen B: Servers (Configurations)

**Purpose:** List all imported servers grouped by source. Primary interaction screen for Phase 1.

| Element | Specification |
|---------|--------------|
| App bar | Title: "Servers" — `titleLarge` |
| Server group header | Subscription name or "Manual" — `titleMedium`, `primary` color, 16dp horizontal padding, 8dp vertical padding |
| Server card | Card widget, 12dp border radius, 1dp elevation, `surfaceContainerLow` fill. Internal padding: 16dp all sides. Height: intrinsic (no fixed height). |
| Card layout | Row: [Protocol badge (left) → 8dp gap] [Column: server name `titleMedium` + server address `bodyMedium`/`onSurfaceVariant`] [Checkmark icon if selected (right)] |
| Protocol badge | Container with 4dp vertical / 8dp horizontal padding, 6dp border radius, protocol-specific color (see table above), `labelMedium` white text |
| Selected state | Card border: 2dp solid `primary` color. Trailing checkmark icon: `Icons.check_circle`, `primary` color |
| Tap interaction | Tap card → select as active server. Previous selection deselected. Haptic feedback (`HapticFeedback.selectionClick`). |
| FAB | `FloatingActionButton.extended` — teal `primary` background, white `Icons.add` icon, label "Import Server". On tap: expands to show sub-options. |
| FAB expanded options | Column of 3 mini-FABs above the main FAB, 8dp vertical gap between each: |
| — Clipboard | `Icons.content_paste`, label "Clipboard" — functional in Phase 1 |
| — Paste Config | `Icons.edit_note`, label "Paste Config" — opens full-screen dialog for manual share link or JSON paste. Functional in Phase 1. |
| — Scan QR | `Icons.qr_code_scanner`, label "Scan QR" — tap shows snackbar "QR scanning coming soon" (Phase 3) |
| Empty state | Centered column: `Icons.dns_outlined` at 64dp size in `onSurfaceVariant` at 40% opacity, 16dp gap, heading "No servers yet", 8dp gap, body "Import your first server config to get started", 24dp gap, `FilledButton` "Import from Clipboard" |

### Screen C: Routing

**Purpose:** Routing rules overview. In Phase 1, shows basic toggle placeholders. Fully functional in Phase 4.

| Element | Specification |
|---------|--------------|
| App bar | Title: "Routing" — `titleLarge` |
| Content | List of `SwitchListTile` widgets with `titleMedium` titles and `bodyMedium` subtitles |
| Bypass LAN toggle | Title: "Bypass LAN", Subtitle: "Don't route local network traffic (192.168.x.x, 10.x.x.x) through proxy", Default: ON |
| Placeholder section | Card with "Custom routing rules coming in a future update" — `bodyMedium`, `onSurfaceVariant`, centered, 48dp vertical padding |

### Screen D: Settings

**Purpose:** App preferences. Theme and language are functional in Phase 1.

| Element | Specification |
|---------|--------------|
| App bar | Title: "Settings" — `titleLarge` |
| Section header | "General" — `labelLarge`, `primary` color, 16dp left padding, 8dp vertical padding |
| Theme selector | `ListTile` with `Icons.brightness_6` leading icon. Title: "Theme" — `titleMedium`. Trailing: `SegmentedButton` with 3 options: "System", "Light", "Dark". Default: "System". |
| Language selector | `ListTile` with `Icons.language` leading icon. Title: "Language" — `titleMedium`. Trailing: current language name. Tap: opens `showModalBottomSheet` with language list. |
| Language options | English, فارسی (Persian), Русский (Russian), 中文 (Chinese). Each as `ListTile` with radio indicator. |
| About section | Section header: "About". `ListTile`: "Version" — `titleMedium`, trailing: "1.0.0" — `bodyMedium`. `ListTile`: "Open Source Licenses". |

### Navigation Shell

| Element | Specification |
|---------|--------------|
| Bottom nav bar | `NavigationBar` (Material 3). 4 destinations. `surfaceContainerLow` background. |
| Tab 1 | Icon: `Icons.dashboard_outlined` / `Icons.dashboard`, Label: "Dashboard" |

## Interaction Patterns

### Feedback Approach

| Interaction | Feedback Type | Specification |
|-------------|--------------|---------------|
| Import success | SnackBar (bottom) | Green tint, "Server imported successfully" + server name, 4 second duration, "View" action |
| Import failure / parse error | SnackBar (bottom) | Error tint (`errorContainer`), error message, 6 second duration, "Retry" action |
| Select server | Haptic + visual | `HapticFeedback.selectionClick()`, card border animates to `primary`, checkmark fades in (200ms) |
| Placeholder tap (disabled connect, QR) | SnackBar (bottom) | Default style, descriptive message, 3 second duration, no action |
| Delete server | Dialog (modal) | Confirmation dialog before deletion (see Destructive below) |
| Theme change | Immediate | No snackbar. Theme applies instantly via `ThemeMode` switch. |
| Language change | Immediate | Locale switches immediately. Snackbar: "Language changed to {language}" |

### Error Handling Pattern

**SnackBar for non-blocking feedback. Dialog ONLY for destructive confirmations.** No toast package needed — use Material 3 `ScaffoldMessenger.showSnackBar()`.

## Paste Config Dialog

Full-screen dialog (`showDialog` with `Dialog.fullscreen` or `Navigator.push` with `MaterialPageRoute`):

| Element | Specification |
|---------|--------------|
| App bar | Leading: "✕" close button. Title: "Paste Config" — `titleLarge`. Trailing: "Import Config" text button (disabled until input is non-empty). |
| Input field | `TextField` with `maxLines: null` (multiline). Hint text from copywriting contract. `OutlineInputBorder` with 12dp radius. Autofocus: true. |
| Validation | On "Import Config" tap: detect format (share link vs JSON). Parse. If valid → pop dialog, show success snackbar, add to server list. If invalid → show inline error below TextField (`errorText` property). |

## Animation Contract

Minimal animations for Phase 1. Declarative via `flutter_animate` or implicit animations:

| Element | Animation | Duration | Curve |
|---------|-----------|----------|-------|
| Server card selection | Border color + checkmark fade in | 200ms | `Curves.easeInOut` |
| FAB expansion | Sub-items scale + fade in (staggered 50ms) | 200ms per item | `Curves.easeOut` |
| FAB collapse | Sub-items fade out | 150ms | `Curves.easeIn` |
| Screen transitions | Material 3 `FadeThrough` (via go_router page transitions) | 300ms | M3 default |
| Theme switch | `AnimatedTheme` wrapper | 300ms | `Curves.easeInOut` |

## Responsive Layout

**Phase 1 is Android-only.** Design for portrait phone layout (360-412dp width typical).

| Breakpoint | Behavior |
|------------|----------|
| < 360dp | Content padding reduces to 8dp. Cards go edge-to-edge with 8dp horizontal margin. |
| 360-600dp | Default layout. 16dp content padding. Standard card layout. |
| > 600dp (tablet) | Not targeted in Phase 1. Same phone layout will display. No tablet-specific adaptations. |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Touch targets | Minimum 48dp × 48dp for all interactive elements |
| Contrast ratio | Material 3 `ColorScheme.fromSeed` ensures WCAG AA compliance by default |
| Screen reader | All interactive elements get `Semantics` labels. Protocol badges: "Protocol: VLESS". Server cards: "{serverName}, {protocol}, {tap to select}" |
| RTL support | Full RTL layout when Persian (fa) locale is active. Test: icons flip, text alignment flips, card layout mirrors |
