<objective>
Integrate the pre-built Xray-core AAR binary, configure Gradle for Go native libraries, set up all Android permissions and VpnService manifest declaration, and bundle geo-data routing assets.

Purpose: This is the native foundation for the entire VPN engine. Without the AAR, no VPN functionality is possible. Without the manifest changes, the VpnService cannot run. Without geo data, routing rules (LAN bypass, split DNS) cannot function.

Output: A compilable Android project with Xray-core AAR recognized by Gradle, VpnService declared in a separate process with Android 14+ permissions, and geo-data assets bundled.
</objective>



**Step 3 — Configure build.gradle.kts:**
Add to the existing `android { }` block, AFTER the `buildTypes { }` section:


```kotlin
dependencies {
    implementation(fileTree(mapOf("dir" to "libs", "include" to listOf("*.aar"))))
```

The final build.gradle.kts should have these exact sections added — keep ALL existing content unchanged.

**Step 1 — Add permissions to AndroidManifest.xml:**
Add these permission declarations BEFORE the `<application>` tag:
```xml
    <uses-permission android:name="android.permission.INTERNET" />
    <uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />
    <uses-permission android:name="android.permission.CHANGE_NETWORK_STATE" />
    <uses-permission android:name="android.permission.FOREGROUND_SERVICE" />
    <uses-permission android:name="android.permission.FOREGROUND_SERVICE_SPECIAL_USE" />
    <uses-permission android:name="android.permission.POST_NOTIFICATIONS" />
```

**Step 2 — Add VpnService declaration inside `<application>` tag:**
After the existing `<activity>` block and BEFORE the `<meta-data android:name="flutterEmbedding"` line, add:
```xml
        <!-- VPN Service — SEPARATE PROCESS (D-06) -->
        <service
            android:name=".service.ArmaVpnService"
            android:process=":vpn_process"
            android:permission="android.permission.BIND_VPN_SERVICE"
            android:foregroundServiceType="specialUse"
            android:exported="false">
            <intent-filter>
                <action android:name="android.net.VpnService" />
            </intent-filter>
            <meta-data
                android:name="android.net.VpnService.SUPPORTS_ALWAYS_ON"
                android:value="true" />
            <property
                android:name="android.app.PROPERTY_SPECIAL_USE_FGS_SUBTYPE"
                android:value="vpn" />
        </service>
```

Keep ALL existing content in the manifest unchanged.
