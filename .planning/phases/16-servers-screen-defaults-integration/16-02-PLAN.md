<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API key payload (`keyBody`) → parser/mapper | Untrusted remote payload is transformed into UI-connectable server rows |
| UI tap targets → connection provider | User-driven selection can trigger disconnect/connect state transitions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-05 | T (Tampering) | `default_server_item_mapper.dart` | mitigate | Parse via existing `SubscriptionParser` + protocol parsers only; skip malformed rows and keep non-connectable fallback instead of blindly trusting payload text. |
| T-16-06 | D (Denial of Service) | `server_list_screen.dart` empty-state gating | mitigate | Keep rendering conditional bounded to provider state and existing list pipeline; protect against loop/rebuild regressions with focused widget tests for empty/imported/default permutations. |
| T-16-07 | S (Spoofing) | Default row label rendering | mitigate | Bind row identity to deterministic IDs and parsed server config IDs so selection/connect actions target actual parsed servers, not display-only labels. |
| T-16-08 | I (Information Disclosure) | Defaults section empty/offline messages | accept | Section text remains generic status copy with no credentials/PII exposure. |
</threat_model>
