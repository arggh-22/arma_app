# Phase 4 — UI Design Contract

> Visual and interaction contract for Routing, DNS & Advanced Settings.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Flutter Material 3 (existing — no changes) |
| Preset | `ColorScheme.fromSeed(seedColor: Color(0xFF00897B))` — Teal 600 |
| Component library | Material 3 widgets (`SwitchListTile`, `SegmentedButton`, `Slider`, `FilterChip`, `ExpansionTile`, `AlertDialog`, `SearchBar`, etc.) |
| Icon library | Material Icons (built-in `Icons.*`) |
| Font | System default (Roboto on Android) via Material 3 TextTheme |

**Source:** `lib/core/theme/app_theme.dart` — established in Phase 1, unchanged across Phases 1–3.

---

## Spacing Scale

All values in logical pixels (dp). 4-point grid, Material 3 conventions. Identical to Phases 1–3.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4dp | Icon-to-label gaps, inline padding, chip internal padding |
| sm | 8dp | Compact element spacing, between list tiles, internal row gaps |
| md | 16dp | Default content padding, card internal padding, horizontal screen margins, section header padding |
| lg | 24dp | Section gaps, between major content groups |
| xl | 32dp | Empty state horizontal padding, page-level content inset |
| 2xl | 48dp | Empty state vertical centering padding |
| 3xl | 64dp | Empty state icon size |

Exceptions:
- Touch targets: minimum 48dp × 48dp (Material accessibility guideline)
- App picker list item height: 56dp (icon + text + checkbox, consistent with Material ListTile default)
- Slider track: 36dp vertical touch area (M3 default Slider)
- Bottom padding on scrollable lists: 88dp (FAB clearance, established pattern)

---

## Typography

All typography uses Material 3 `TextTheme` roles — no custom font sizes. Continues Phase 1–3 conventions exactly.

| Role | M3 TextTheme | Size | Weight | Line Height | Usage in Phase 4 |
|------|-------------|------|--------|-------------|-------------------|
| Body | `bodyMedium` | 14sp | 400 (regular) | 1.43 (20sp) | Setting descriptions, subtitle text, DNS URL display, domain entries, app names |
| Body small | `bodySmall` | 12sp | 400 (regular) | 1.33 (16sp) | Helper text, preset descriptions, field hints, rule count badges |
| Label | `labelLarge` | 14sp | 500 (medium) | 1.43 (20sp) | Section headers (with `primary` color), chip labels, action button text |
| Title | `titleMedium` | 16sp | 500 (medium) | 1.5 (24sp) | ListTile titles, dialog titles, sub-screen AppBar titles |
| Heading | `headlineSmall` | 24sp | 400 (regular) | 1.33 (32sp) | Empty state headings |
| App bar | `titleLarge` | 22sp | 400 (regular) | 1.27 (28sp) | Screen titles in AppBar |

**Weight constraint:** Only 2 weights: **400 (regular)** and **500 (medium)**. No bold, no light. Matches Phases 1–3.

---

## Color

Inherits Material 3 dynamic color scheme from `ColorScheme.fromSeed(seedColor: Color(0xFF00897B))`. No new custom colors introduced.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `colorScheme.surface` | Scaffold backgrounds, list backgrounds |
| Secondary (30%) | `colorScheme.surfaceContainerLow` | Cards, ExpansionTile backgrounds, app picker item backgrounds, preset cards |
| Accent (10%) | `colorScheme.primary` | See reserved-for list below |
| Destructive | `colorScheme.error` | Delete rule, clear cache confirmation, error states |

**Accent reserved for (Phase 4 additions only):**
1. Active SegmentedButton selection (Blacklist/Whitelist toggle, DNS protocol selector)
2. Region preset chip selected state (filled background)
3. Section header text ("Engine Settings", "Anti-Censorship", "DNS", "Domain Rules", "Per-App Proxy")
4. Profile card active indicator (Light/Moderate/Aggressive anti-censorship profile)
5. Switch/toggle active state (sniffing, mux, fragment toggles)
6. Save/Apply action buttons (FilledButton backgrounds)
7. Add rule FAB on routing screen

**Routing action colors (semantic):**

| Action | Color | Token |
|--------|-------|-------|
| Proxy | Teal (accent) | `colorScheme.primary` |
| Direct | Green | `Colors.green` (0xFF4CAF50) |
| Block | Red | `colorScheme.error` |

These colors are used as subtle background tints on domain rule action chips and as leading dot indicators on rule list items.

## Interaction Contracts

### Region Presets (D-01, D-02)

| Trigger | Action |
|---------|--------|
| Tap region chip (unselected) | Chip fills with `primary` color. Region rules become active. Xray routing config regenerates on next connect. |
| Tap region chip (selected) | Chip returns to unselected state. Region rules deactivated. |
| Tap "Update Rules" | Fetches latest community rules from GitHub (chocolate4u/Iran-v2ray-rules etc.). Shows progress indicator. On success: SnackBar "Rules updated". On fail: SnackBar with error. |
| Multiple regions | Can enable multiple regions simultaneously. Rules merge (union). |

### Domain Rules (D-03)

| Trigger | Action |
|---------|--------|
| Tap "Add Rule" | Opens AlertDialog with domain field + action SegmentedButton |
| Submit valid domain | Rule appears in list immediately. Persisted to Hive. Dialog closes. |
| Submit invalid domain | Inline validation error below TextField: "Enter a valid domain (e.g., example.com)" |
| Change action dropdown | Updates rule action immediately. Persisted. No confirmation. |
| Tap delete (✕) on rule | Rule removed immediately. SnackBar "Rule removed" with "Undo" action (3 second window). |
| Undo delete | Rule restored to its previous position in the list. |

### Per-App Proxy (D-04, D-05)

| Trigger | Action |
|---------|--------|
| Toggle "Enable Per-App Proxy" ON | App list loads and becomes visible. Android `PackageManager` enumeration runs async. |
| Toggle "Enable Per-App Proxy" OFF | App list hides. Per-app routing disabled — all apps go through VPN. |
| Switch Blacklist ↔ Whitelist | Mode switches. **Selection clears** to prevent confusion. SnackBar: "Switched to {mode} mode". |
| Tap checkbox on app | App added/removed from selection set. Persisted immediately. |
| Type in search bar | App list filters by name (case-insensitive substring match). |
| Clear search | Full list restored. |

### Anti-Censorship Profiles (D-11)

| Trigger | Action |
|---------|--------|
| Select profile "None" | All anti-censorship toggles reset to OFF. Fields reset to empty/default. |
| Select profile "Light" | Auto-fills: Fragment ON, size 10–50, sleep 0–0, padding OFF, mixed SNI OFF. |
| Select profile "Moderate" | Auto-fills: Fragment ON, size 10–100, sleep 1–10, padding ON, mixed SNI OFF. |
| Select profile "Aggressive" | Auto-fills: Fragment ON, size 1–100, sleep 10–50, padding ON, mixed SNI ON. |
| Manually change any field after profile selection | Profile indicator remains but user's customization takes precedence. No auto-revert. |
