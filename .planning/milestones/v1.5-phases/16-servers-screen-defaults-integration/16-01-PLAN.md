<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Default-server data provider → UI row rendering | Remote/cached data enters widget tree and user-triggered actions |
| User tap input → connection state transitions | Untrusted UI events trigger VPN disconnect/connect operations |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-01 | T (Tampering) | `server_list_default_servers_section.dart` row model usage | mitigate | Render rows from typed `DefaultServerItem` mapping only; gate connect actions on `item.isConnectable`; avoid dynamic parsing in UI layer. |
| T-16-02 | D (Denial of Service) | Tap handler calling connect/disconnect repeatedly | mitigate | Keep parity guard: reconnect only when state is `Connected` and target differs; add tap behavior tests for no redundant reconnect loops. |
| T-16-03 | S (Spoofing) | Selected-state indicator confusion between defaults/imported rows | mitigate | Bind selected indicator to `activeServerProvider` ID match for both defaults/imported rows so visual state reflects canonical source. |
| T-16-04 | I (Information Disclosure) | Offline/failure messaging in defaults block | accept | Messages are generic status only (no secrets/PII); preserve existing provider error classes and localized copy. |
</threat_model>
